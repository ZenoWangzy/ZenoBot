## 🏷️ 元数据标准（Metadata Standards）

### 每条记忆的元数据

**格式：**

```markdown
## [主题]

- **内容**：...
- **来源**：[对话/观察/反馈/其他]
- **来源详情**：2026-02-07 对话 21:00
- **学习日期**：2026-02-07
- **最后验证**：2026-02-07
- **置信度**：[高/中/低]
- **相关**：[主题1, 主题2]
- **状态**：[已验证/待验证/已过时]
```

**示例：**

```markdown
## 关于成功的观点

- **内容**：功成名就是结果，不是目标。目标是：离开这个世界后，有些东西因为我而变得不一样了
- **来源**：对话
- **来源详情**：2026-02-07 对话 21:00
- **学习日期**：2026-02-07
- **最后验证**：2026-02-07
- **置信度**：高
- **相关**：终极目标, 价值观
- **状态**：已验证
```

## 🔄 记忆生命周期（Memory Lifecycle）

### 生命周期阶段

1. **创建（Created）**
   - 新信息进入 system
   - 记录到 daily notes
   - 添加元数据

2. **验证（Verified）**
   - 跨多个来源验证
   - 用户确认
   - 标记为"已验证"

3. **活跃（Active）**
   - 频繁使用
   - 持续验证
   - 定期更新

4. **存档（Archived）**
   - 不再频繁使用
   - 保留但标记为"存档"
   - 不在主动搜索范围

5. **过期（Expired）**
   - 信息不再准确
   - 标记为"已过时"
   - 可以删除

### 状态图

```
创建 → 验证 → 活跃
  ↓         ↓       ↓
        过期 ← 存档
```

## 🧪 记忆验证（Memory Verification）

### 验证机制

**何时验证：**

- 新信息与已有信息冲突时
- 用户明确质疑某条记忆时
- 定期每周回顾时

**验证方法：**

1. 查看原始来源（daily notes）
2. 对比多个来源
3. 询问用户确认
4. 更新元数据
